# Industrial Ops Brain

Local evidence workbench for ET AI Hackathon 2026. The app runs on a local Next.js frontend, a local FastAPI backend, local Postgres with pgvector, local file uploads, local OCR/parsing, and FastEmbed retrieval embeddings.

The only intended external runtime request is the DeepSeek chat completions API when `DEEPSEEK_API_KEY` is configured.

## Local Setup

1. Install Python 3.11, Node.js 26, npm, and a local Postgres instance with pgvector available.
2. Create local databases matching `.env.example`, or edit `DATABASE_URL` and `TEST_DATABASE_URL`.
3. Set `DEEPSEEK_API_KEY` in `.env` if generated analysis or chat answers should call DeepSeek.
4. Run setup:

```powershell
npm run setup
```

5. Start both local services:

```powershell
npm run dev
```

Local URLs:

- Frontend: `http://localhost:3000`
- Backend health: `http://127.0.0.1:8000/health`
- Backend API docs: `http://127.0.0.1:8000/docs`

## Environment

Required local settings:

- `DATABASE_URL`: local Postgres URL for app data.
- `TEST_DATABASE_URL`: local Postgres URL for tests and smoke checks.
- `DEEPSEEK_API_KEY`: optional, required only for live generated analysis and chat.
- `DEEPSEEK_BASE_URL`: defaults to `https://api.deepseek.com`.
- `DEEPSEEK_MODEL`: defaults to `deepseek-v4-flash`.
- `EMBEDDING_MODEL`: defaults to `BAAI/bge-small-en-v1.5`.
- `EMBEDDING_CACHE_DIR`: local FastEmbed model cache, normally `backend/data/fastembed`.
- `NEXT_PUBLIC_API_BASE_URL`: local backend URL, normally `http://127.0.0.1:8000`.

## Runtime Flow

- Upload evidence through `POST /documents/upload-batch`.
- Files are validated, stored under `backend/data/uploads`, parsed, chunked, and indexed inline.
- Embeddings are generated by FastEmbed in `backend/app/services/embeddings.py` and stored in `chunks.embedding` as pgvector values.
- Chat and RCA embed the query, retrieve nearest uploaded chunks from pgvector, and pass only that cited context to DeepSeek.
- Analysis and chat call DeepSeek only when configured.
- `DELETE /workspace` clears local workspace records and tracked local upload files.

## Verification

```powershell
npm run frontend:typecheck
npm run frontend:lint
npm run backend:test
```

Use `npm run smoke` when `TEST_DATABASE_URL` points to a disposable local Postgres database.
